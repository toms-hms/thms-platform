# API

Express + TypeScript + Drizzle ORM + PostgreSQL.

## Module structure (enforced)
```
src/{module}/
  models/
    ModelName.ts         Drizzle table definition + inferred types
    ModelNameManager.ts  Query methods (Django-manager style)
  route.ts               Router only ‚Äî no business logic
  schema.ts              Zod schemas only
  service.ts             Thin orchestration ‚Äî calls managers
src/db/
  index.ts               Single Drizzle client (import { db } from '../db')
  schema.ts              Re-exports all tables for drizzle-kit
  enums.ts               All pgEnum definitions
```
Modules: `auth`, `home`, `job`, `contractor`, `quote`, `communication`, `integration`, `ai`, `upload`

## Non-obvious patterns
- Route auth cast: `(req as unknown as AuthenticatedRequest).user.userId`
- `job/route.ts` exports TWO routers: `jobCollectionRouter` + `jobRouter`
- IDs generated with `createId()` from `@paralleldrive/cuid2` ‚Äî NOT auto-generated by DB
- Always pass `id`, `updatedAt: new Date()` when inserting
- Test cleanup: delete homes before users (Job‚ÜíUser FK has no cascade)
- `z.string().cuid()` validators replaced with `z.string().min(1)` ‚Äî IDs are now CUID2 format
- Tests mock `openai` and `googleapis` via `src/__tests__/__mocks__/`
- Run tests: `NODE_OPTIONS="--max-old-space-size=3072" npx jest --runInBand --forceExit`

## DB migrations (Drizzle)
```bash
make migrate    # generate SQL + apply
make studio     # open Drizzle Studio
```
Baseline migration already applied. All future changes: edit model .ts file ‚Üí `make migrate`.
