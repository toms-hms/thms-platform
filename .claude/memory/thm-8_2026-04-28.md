**Key decisions:**
- `category` column replaced by `categories tradeCategoryEnum('categories').array().notNull()` directly on the `Contractor` row — no join table (enum array is the right pattern; a join table M2M makes no sense when one side is an enum, not a table)
- `ContractorZipCode` is a separate table with its own `id` PK because zip codes are real data rows
- `ContractorManager.create(data, zipCodes)` — categories go in the data object, zipCodes separate
- `ContractorManager.update(id, data, zipCodes?)` — omitting `zipCodes` leaves existing values unchanged; categories updated via normal column set
- Migration required: run `make migrate` to generate and apply the schema diff
